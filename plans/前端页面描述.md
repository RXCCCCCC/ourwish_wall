📷 页面设计描述提示词 (Copy below)
角色设定：你是一名精通 Vue 3、Tailwind CSS 和现代 UI 设计的前端工程师。

任务：请根据以下详细的视觉描述，复刻“数字传承 · 红色德兴”的心愿墙网页前端代码。

1. 整体布局与风格
背景：整体使用极简的亮白色背景 (#FFFFFF 或非常淡的灰白色 #F9FAFB)。

设计语言：现代、庄重、具有“红色文化”氛围但不俗气。使用大量留白（Whitespace）来突出内容。

字体：标题使用衬线体（宋体/Serif）以体现历史感，正文使用无衬线体（黑体/Sans-serif）以保证易读性。

2. 顶部导航栏 (Navbar)
布局：Flexbox 左右两端对齐 (justify-between)，高度约 80px。

左侧 Logo 区域：

图标：一个圆角的红色正方形背景 (#E60012)，中间有一颗白色的五角星。

主标题：文字“数字传承 · 红色德兴”，字体加粗，黑色。

副标题：下方紧跟一行微小的英文 "DIGITAL INHERITANCE OF RED GENES"，灰色全大写。

右侧操作区：

按钮：一个胶囊形按钮。背景色为深藏青色/黑色 (#0F172A)，文字为白色“+ 发布心愿”。鼠标悬停时应有轻微的变色效果。

3. 核心宣传区 (Hero Section)
布局：居中对齐 (text-center)，上下有较大的内边距 (py-12 或更多)。

标签 (Tag)：顶部有一个淡红色/粉色的胶囊标签 (bg-red-50 text-red-600)，带有一个小图标（如博物馆或建筑icon），文字为“科技激活德兴文旅”。

主标题：巨大的衬线字体标题“汇聚微光，点亮红土地”，字号应很大 (text-4xl 或更大)，颜色为深黑色。

描述文本：标题下方有一段居中的灰色文本 (text-gray-500)，最大宽度限制在 600px 左右。内容关于“心愿成为数字记忆... AI智能助手连接历史与未来”。

4. 心愿卡片网格 (Wish Cards Grid)
布局：Grid 布局，双列显示（在移动端应折叠为单列）。

卡片通用样式：

白色背景，带有极其轻微的阴影 (shadow-sm)。

圆角 (rounded-xl)。

左侧装饰线：每个卡片最左侧有一条粗边框线（约 4px 宽）。左边卡片为红色 (border-l-red-600)，右边卡片为深蓝色 (border-l-slate-700)。

内边距：充足的 Padding (p-6)。

卡片内部元素：

顶部日期：左上角显示日期“2026年1月22日”，日期前有一个对应卡片颜色（红/蓝）的小圆点。

引用图标：右上角有一个巨大的、淡灰色的双引号图标 (") 作为装饰。

心愿内容：

左卡片：使用手写体/楷体风格，内容为“愿德兴的红色故事代代相传...”。

右卡片：使用标准黑体，内容为“希望能用VR技术复原...”。

AI 回复框 (Digital Echo)：

位于心愿内容下方。

背景为极淡的灰色或粉色 (bg-gray-50)，圆角。

标题：带有一个红色小图标（如星星或火花） + 文字“数字回响”，红色字体。

正文：灰色的 AI 生成回复文本，字体较小。

底部用户信息：

左侧：圆形头像（带文字） + 用户昵称（如“张建国”）。

右侧：爱心图标（空心或实心） + 点赞数（灰色数字）。

5. 技术栈要求
框架：Vue 3 (Script Setup)。

样式：Tailwind CSS (或者使用纯 CSS / SCSS)。

图标库：可以使用 Lucide-vue 或 Heroicons。

6. 新增交互功能（更新 2026-01-22）

为了增强用户互动与社区感，前端实现并预留对接后端的以下功能：

- **点赞（所有用户）**：每条心愿支持点赞，点赞计数显示在卡片右下方。前端使用 `localStorage` 记录当前用户是否已对某条心愿点赞以避免重复点赞。后端接口建议：`POST /api/wishes/:id/like`（用于持久化并返回最新点赞数）。

- **评论（所有用户）**：支持对心愿发表评论，评论数据结构包含 `id, content, nickname, avatar, user_uid, created_at`。前端会在提交后立即显示（乐观更新），后端接口建议：`POST /api/wishes/:id/comments`。

- **删除评论（评论作者或管理员）**：只有评论的作者（与 `user_uid` 匹配的当前用户）或管理员可删除评论。前端显示“删除”按钮仅在作者身份匹配时可见。后端接口建议：`DELETE /api/wishes/:id/comments/:comment_id`。

- **删除心愿（仅发布者）**：心愿发布者在自己的心愿卡上可见“删除心愿”按钮，确认后可删除。后端接口建议：`DELETE /api/wishes/:id`（执行权限检查，确保仅发布者可删除）。

注意：前端当前实现了乐观更新逻辑（立即在 UI 显示操作结果），请在后端实现以上接口后将前端注释中的 TODO 点替换为实际网络请求以确保数据持久化与一致性。
