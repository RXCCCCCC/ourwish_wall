# 德兴“红脉新声”心愿云墙 H5 系统设计方案 (公网版)

## 1. 核心变化与策略调整

* **平台形态**：**H5 移动端网页**（运行在微信浏览器中）。
* *优势*：开发一次，处处运行；无需微信小程序账号审核，无类目限制。
* *注意*：因为不是原生小程序，无法直接静默调用“微信一键登录”获取头像昵称（需要企业认证的服务号）。
* *解决方案*：设计一个**“伪登录”页面**，用户进入时随机生成一个“红色代号”（如：红军战士_007）和头像，或者允许用户手动输入昵称，体验上非常接近小程序。


* **部署方式**：**云服务器部署**（非局域网）。
* *必须条件*：你需要一台轻量级云服务器（推荐腾讯云/阿里云/华为云的学生机，约 ¥9-30/月）和一个域名（可选，直接用IP也能跑，但微信可能会提示“不安全”）。


* **技术栈**：Vue 3 (H5模式) + Flask (API) + MySQL + ECharts (可视化)。

---

## 2. 系统架构设计

### 2.1 前端 (Vue 3 + Vant 4)

* **入口**：用户扫描二维码 -> 打开 URL (http://你的公网IP)。
* **页面规划**：
1. **启动/授权页**：检测本地缓存，如果没有用户信息，弹窗让用户输入昵称或“一键生成身份”。
2. **心愿广场 (主页)**：顶部展示**数据大屏** (词云+饼图)，下方瀑布流展示心愿。
3. **许愿页**：悬浮按钮点击进入，表单提交。



### 2.2 后端 (Flask + PyMySQL)

* 部署在云服务器的 Linux 系统上（Ubuntu/CentOS）。
* 使用 **Gunicorn** 作为 WSGI 服务器，**Nginx** 作为反向代理（处理静态文件和转发 API）。

## 3. 功能模块详细设计

### 模块一：用户身份系统 (H5 适配版)

由于无法直接调用微信 `wx.getUserProfile`，我们采用**“轻量级身份标记”**策略。

* **逻辑**：
1. App 启动时检查 `localStorage` 是否有 `user_token`。
2. 若无，跳转到“欢迎页”。
3. 用户点击“加入红脉行动”，前端自动生成一个 UUID 和随机昵称（或用户手填），存入 `localStorage` 并发送给后端注册。


* **代码思路 (Vue)**:
```javascript
// store/user.js
export const useUserStore = defineStore('user', {
  state: () => ({ userInfo: JSON.parse(localStorage.getItem('user_info')) || null }),
  actions: {
    login(nickname) {
      // 模拟登录，实际项目中这里调用后端API注册
      this.userInfo = { 
        id: Date.now(), 
        name: nickname || `红脉卫士${Math.floor(Math.random()*1000)}`,
        avatar: 'default_red_avatar.png'
      };
      localStorage.setItem('user_info', JSON.stringify(this.userInfo));
    }
  }
});

```



### 模块二：数据可视化 (前端直接渲染)

需求要求“词云和饼图保留并呈现在前端”。

* **技术选型**：ECharts (饼图) + echarts-wordcloud (词云)。
* **布局位置**：放在“心愿广场”页面的顶部 Header 区域，作为“实时战报”展示。

## 5. 公网部署指南 (关键步骤)

这是实现“何时何地都能访问”的关键。

### 第一步：购买与配置云服务器

1. 购买腾讯云/阿里云 **轻量应用服务器** (Lightweight Server)。
* 镜像选择：**Docker CE** (最简单，环境全包) 或者 **Ubuntu 20.04**。


2. 在云控制台的**防火墙/安全组**中，开放端口：
* `80` (HTTP)
* `5000` (Flask后端，或者用Nginx转发)
* `3306` (MySQL，建议仅限本机访问)



### 第二步：后端部署 (Flask)

1. SSH 连接到服务器。
2. 安装 Python3 和依赖：
```bash
pip3 install flask pymysql flask-cors gunicorn

```


3. 上传你的 Python 代码 (使用 FileZilla 或 scp)。
4. 后台运行 Flask：
```bash
# 使用 gunicorn 运行，绑定 0.0.0.0:5000
nohup gunicorn -w 4 -b 0.0.0.0:5000 app:app &

```



### 第三步：前端部署 (Vue build)

1. 在本地修改 Vue 项目的 API 地址：
* 把 `axios.defaults.baseURL` 从 `localhost` 改为 `http://你的公网IP:5000`。


2. 打包项目：
```bash
npm run build

```


3. 将生成的 `dist` 文件夹上传到服务器的 `/var/www/html` (假设使用 Nginx)。

### 第四步：Nginx 配置 (反向代理)

配置 Nginx 让用户访问 80 端口就能看到网页，且解决跨域问题。

```nginx
server {
    listen 80;
    server_name _; # 你的公网IP

    # 前端页面
    location / {
        root /var/www/html/dist;
        index index.html;
        try_files $uri $uri/ /index.html; # Vue Router History模式必加
    }

    # 后端接口转发
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
    }
}

```

## 7. 风险提示与对策

1. **微信拦截风险**：微信对于只有 IP 地址（没有域名）的网页，有时会提示“不安全”或限制访问。
* *对策*：如果只是短期实践，忽略警告直接点“继续访问”即可。如果想完美，需要买个域名（.top/.xyz 很便宜）并备案（备案通过需要时间，可能来不及）。**建议直接用 IP 访问，或者生成草料二维码时选择“活码”，万一被封可以换链接。**


2. **网络图片加载**：公网访问时，用户上传的图片如果太大，加载会很慢。
* *对策*：前端上传前必须压缩图片（使用 `compressorjs` 库），限制在 200KB 以内。



这个方案砍掉了最复杂的 APP 打包，用 Web 技术解决了一切，同时通过云部署满足了远程交互的需求，是目前性价比最高的路径。
